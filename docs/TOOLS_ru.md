# Описание инструментов и скриптов проекта

[ English ](TOOLS.md) | [ Русский ](TOOLS_ru.md)

Этот документ описывает назначение и использование скриптов, расположенных в директории `tools/`.

## Директория tools/

### `build_matrix.sh`
Автоматизирует сборку модуля ядра для нескольких версий ядер. Использует переменную окружения `KERNELDIRS` для поиска деревьев ядер.
- **Использование:** `export KERNELDIRS="/path/to/k1 /path/to/k2"; ./tools/build_matrix.sh`
- **Особенности:** По умолчанию выполняет чистую сборку, записывает результаты в `logs/`, продолжает работу при ошибках, если `KEEP_GOING=1`.

### `build_one.sh`
Вспомогательный скрипт для сборки модуля для одного конкретного ядра.
- **Использование:** `KERNELDIR=/path/to/kernel ./tools/build_one.sh`

### `prepare_kernels_arm64.sh`
Комплексная утилита для загрузки и подготовки заголовочных файлов / деревьев сборки ядер для различных архитектур и версий.
- **Использование:** Подробные инструкции см. в [prepare_kernels_ru.md](prepare_kernels_ru.md).
- **Особенности:** Идемпотентность, поддержка кросс-компиляции, управление файлом `releases.json`.

### `txgbe_client_dual_iperf_bound.sh`
Скрипт на стороне клиента для детерминированного тестирования `iperf3` на двух портах. Привязывает каждый экземпляр iperf к конкретному исходному IP и назначению.
- **Использование:** Используется для проверки производительности на двухпортовых сетевых картах.

### `txgbe_server_dual_iperf_bound.sh`
Серверная часть для `txgbe_client_dual_iperf_bound.sh`. Запускает два сервера `iperf3`, привязанных к разным IP.

### `verify_kcompat_switch.sh`
Утилита для проверки корректности перегенерации определений `kcompat` при переключении между различными версиями ядер.

### `first_errors.sh`
Простой помощник для извлечения первых нескольких ошибок из лога сборки.

### `releases.json`
Не скрипт, а конфигурационный файл, содержащий зафиксированные версии ядер для утилиты подготовки.

## Директория tools/test/

### `txgbe_client_dual_iperf_bound_v2.sh` / `v3.sh`
Развивающиеся версии клиентского скрипта iperf для двух портов с улучшенной логикой или параметрами.

### `txgbe_loongarch_collect_diag_v2.sh`
Специальный скрипт сбора диагностических данных для систем Loongson-3A6000. Собирает сопоставления IRQ, статистику ethtool и dmesg.

### `txgbe_loongarch_tune_v2.sh`
Скрипт настройки производительности для систем Loongson. Устанавливает RSS, сходство IRQ и другие параметры модуля.

### `txgbe_server_iperf_config_fixed_v2.sh`
Обновленный скрипт конфигурации iperf на стороне сервера.
