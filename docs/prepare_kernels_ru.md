# Утилита подготовки заголовочных файлов ядра

[ English ](prepare_kernels.md) | [ Русский ](prepare_kernels_ru.md)

Эта утилита подготавливает **деревья сборки ядра Linux** для **компиляции внешних модулей** с помощью `modules_prepare` для **нескольких версий ядер** и **нескольких архитектур**.

Она предназначена для:
- разработчиков драйверов
- конвейеров CI (непрерывной интеграции)
- сред кросс-компиляции
- долгосрочной поддержки нескольких серий ядер (включая ядра EOL)

Скрипт является **идемпотентным**, **детерминированным по умолчанию** и поддерживает опциональный автоматический переход на новые версии ядер, если это явно включено.

---

## Что делает этот скрипт

Для каждой настроенной версии ядра скрипт:

1. Загружает соответствующий архив исходных кодов ядра с `cdn.kernel.org`.
2. Проверяет целостность архива (`tar -tf`).
3. Распаковывает исходные коды ядра.
4. Подготавливает директорию сборки (`O=...`) для внешних модулей:
   - импортирует `.config`, если он предоставлен;
   - запускает `defconfig`, если необходимо;
   - запускает `olddefconfig`;
   - запускает `modules_prepare`.
5. Кэширует все результаты и **никогда не скачивает и не пересобирает заново**, если это не запрошено явно.

Результатом является готовая к использованию директория `KERNELDIR`, подходящая для:
```bash
make -C $KERNELDIR M=/путь/к/модулю modules
```

---

## Структура директорий

По умолчанию все ядра подготавливаются в:

```
/opt/kernels/<версия-ядра>/
 ├── _dl/               # загруженные архивы
 ├── src/
 │    └── linux-<вер>/  # распакованные исходники ядра
 └── build-<ARCH>/      # подготовленная директория сборки (KERNELDIR)
```

Пример:
```
/opt/kernels/5.15.198/build-arm64
```

---

## Требования

* Debian 12/13 (или совместимый)
* Права root (запись в `/opt`)
* Инструменты:
  * `gcc` или кросс-компилятор
  * `make`, `tar`, `curl`, `jq`
  * зависимости для сборки ядра (`bc`, `bison`, `flex`, `libelf-dev` и т. д.)

Пример:
```bash
apt install build-essential bc bison flex libelf-dev jq curl
```

## Файл фиксации версий ядра (`releases.json`)

Версии ядер определяются через **локальный файл фиксации**:

```json
{
  "kernels": [
    { "series": "4.19", "version": "4.19.325" },
    { "series": "5.4",  "version": "5.4.287" },
    { "series": "5.10", "version": "5.10.221" },
    { "series": "5.15", "version": "5.15.166" },
    { "series": "6.1",  "version": "6.1.78" },
    { "series": "6.6",  "version": "6.6.63" },
    { "series": "6.12", "version": "6.12.14" },
    { "series": "6.18", "version": "6.18.0" }
  ]
}
```

Это гарантирует **воспроизводимость сборок**, даже если API kernel.org изменится.

## Основное использование

### Подготовка ядер для arm64 (кросс-сборка)

```bash
./prepare_kernels_arm64.sh \
  --arch arm64 \
  --cross-compile aarch64-linux-gnu- \
  --releases-json /etc/kernel-build/releases.json
```

### Подготовка ядер для нативной архитектуры x86_64

```bash
./prepare_kernels_arm64.sh \
  --arch x86_64 \
  --cross-compile "" \
  --releases-json /etc/kernel-build/releases.json
```

---

## Режим имитации (Рекомендуемый первый шаг)

Показать, что **было бы** сделано без изменения системы:

```bash
./prepare_kernels_arm64.sh \
  --arch x86_64 \
  --cross-compile "" \
  --releases-json /etc/kernel-build/releases.json \
  --dry-run
```

Вывод включает:
* выбранные версии ядер;
* URL-адреса архивов;
* локальные пути;
* обнаруженное состояние (загружено / распаковано / подготовлено);
* планируемые действия (`skip / download / extract / prepare`).

---

## Выбор архитектуры

```bash
--arch <arch>
```
Примеры: `arm64`, `x86_64`, `arm`, `riscv`.

Скрипт:
* передает `ARCH=<arch>` в Kbuild;
* проверяет, что `.config` соответствует выбранной архитектуре;
* проверяет, что компилятор соответствует архитектуре.

---

## Выбор инструментария (Toolchain)

```bash
--cross-compile <prefix>
```
Примеры: `aarch64-linux-gnu-`, `arm-linux-gnueabihf-`, `""` (пусто = нативный компилятор).

Скрипт:
* проверяет наличие компилятора;
* выводит `gcc --version`;
* выводит `gcc -dumpmachine`;
* **завершается ошибкой на раннем этапе**, если компилятор и ARCH не совпадают.

---

## Импорт файлов ядра `.config` (опционально)

```bash
--config-dir /путь/к/конфигам
```

Поддерживаемый порядок поиска:
```
<dir>/<ARCH>/<full-version>/.config
<dir>/<ARCH>/<series>/.config
<dir>/<full-version>/.config
<dir>/<series>/.config
```

---

## Обновление до новых версий ядра (опционально)

### Использовать последние доступные для всех серий
```bash
--latest-all
```
* Запрашивает данные с kernel.org.
* Выбирает самую новую доступную версию для каждой серии.
* Проверяет доступность архива на CDN.
* **Не изменяет ваш файл фиксации (lock file).**

---

## Типовой рабочий процесс CI

```bash
# Шаг 1: имитация запуска
./prepare_kernels_arm64.sh --dry-run ...

# Шаг 2: реальная подготовка
./prepare_kernels_arm64.sh ...

# Шаг 3: сборка модуля
make -C /opt/kernels/5.15.198/build-arm64 M=$PWD modules
```
