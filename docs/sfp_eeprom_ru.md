# Как прочитать данные EEPROM SFP (декодированные и сырые) и уровни сигналов

[ English ](sfp_eeprom.md) | [ Русский ](sfp_eeprom_ru.md)

Для работы с EEPROM и диагностикой SFP-модулей в Linux используется стандартная утилита `ethtool` с опцией `-m` (--dump-module-eeprom). Она позволяет как получить "сырой" дамп данных, так и автоматически декодировать их в понятный вид, включая показания уровня сигнала, температуры и напряжения .

Вот как это делается.

### 1. Декодированный вывод (уровень сигнала, температура и т.д.)

Это самый простой и наглядный способ. Просто выполните команду для вашего интерфейса:

```bash
sudo ethtool -m <имя_интерфейса>
```

**Пример вывода:**
В результатах вы увидите информацию о производителе, типе трансивера, а если модуль поддерживает цифровую диагностику (DOM) — то и данные мониторинга .

```text
...
        Vendor SN                                 : F2220644072
        Date code                                 : 220824
        Optical diagnostics support               : Yes
        Laser bias current                        : 6.000 mA
        Laser output power                        : 0.5000 mW / -3.01 dBm
        Receiver signal average optical power     : 0.4000 mW / -3.98 dBm
        Module temperature                        : 50.41 degrees C / 122.75 degrees F
        Module voltage                            : 3.2811 V
        Alarm/warning flags implemented           : Yes
        Laser bias current high alarm threshold   : 15.000 mA
...
```

### 2. "Сырой" дамп EEPROM (прямое шестнадцатеричное представление)

Если необходимо посмотреть данные побайтово, например, для отладки или работы со спецификацией SFF-8472, используйте ключ `raw on` и `hex on` .

```bash
sudo ethtool -m <имя_интерфейса> raw on hex on
```

**Пример вывода:**

```text
Offset          Values

0x0000:         03 04 00 00 00 00 01 00 00 00 00 01 0d 00 00 00
0x0010:         00 00 00 00 46 53 20 20 20 20 20 20 20 20 00 00
0x0020:         00 00 00 00 47 53 46 50 2d 31 47 2d 54 58 20 20
...
```

*   `Offset` — смещение в байтах.
*   `Values` — шестнадцатеричные значения байтов. Расшифровать их можно по спецификациям SFF-8472 (для SFP) или SFF-8636 (для QSFP) .

### 3. Выборочный дамп и диагностика CMIS (для современных модулей)

*   **Чтение определенной области**: Если нужен только фрагмент, например, от `0x14` длиной `32` байта:
    ```bash
    sudo ethtool -m eth0 offset 0x14 length 32
    ```
    В этом примере вывод будет содержать ASCII-строку с именем производителя .

*   **Новые модули (CMIS)**: Современные высокоскоростные модули (400G и выше) используют стандарт CMIS. Современные версии `ethtool` и ядра умеют работать с банками памяти таких модулей, извлекая телеметрию . Команда остается той же: `sudo ethtool -m eth0`.

### Проблемы и их решения

1.  **Нет информации об уровне сигнала (температуре и т.д.):**
    *   Это означает, что либо ваш модуль **не поддерживает функцию цифровой диагностики (DOM)** (часто встречается у дешевых или старых трансиверов), либо он является пассивным медным патч-кордом (DAC), у которого нет электроники для диагностики .
    *   В выводе команды без ключей в секции "Optical diagnostics support" будет указано "No".

2.  **Вывод в виде непонятных "кракозябр" вместо красивого текста:**
    *   В некоторых дистрибутивах, особенно во встраиваемых системах вроде OpenWrt, базовая версия `ethtool` может не иметь поддержки декодирования. В этом случае команда `ethtool -m` выдаст только шестнадцатеричный дамп.
    *   **Решение:** Установите полную версию утилиты. В OpenWrt это пакет `ethtool-full` .
    ```bash
    # Пример для Debian/Ubuntu
    sudo apt-get install ethtool
    # Пример для OpenWrt
    opkg install ethtool-full
    ```

3.  **Нет доступа к SFP (Permission denied):**
    *   Для доступа к диагностическим данным часто требуются права суперпользователя. Всегда используйте `sudo` для выполнения `ethtool -m`.

### Краткий справочник по опциям

| Опция | Назначение | Пример |
| :--- | :--- | :--- |
| **`-m`** | Основная команда для работы с SFP/QSFP модулями | `ethtool -m eth0` |
| **`raw on`** | Вывод данных в сыром, необработанном виде (вместе с `hex on`) | `ethtool -m eth0 raw on` |
| **`hex on`** | Вывод значений в шестнадцатеричном формате | `ethtool -m eth0 hex on` |
| **`offset N`** | Начать чтение со смещения `N` (в байтах) | `ethtool -m eth0 offset 64` |
| **`length N`** | Прочитать `N` байт | `ethtool -m eth0 length 128` |

Эти команды покрывают практически все задачи по мониторингу SFP — от проверки уровня сигнала (TX/RX power) до сбора серийных номеров для инвентаризации.
