# Журнал изменений (CHANGELOG)

[ English ](CHANGELOG.md) | [ Русский ](CHANGELOG_ru.md)

## Невыпущенные изменения

- Добавлена документация по запуску 10G на двух портах Loongson-3A6000 / LoongArch64 и вспомогательные скрипты для детерминированного теста `iperf3`.
- Задокументированы стабильные параметры тестирования для Loongson: `RSS=4,4`, адаптивное ограничение прерываний, принудительная привязка IRQ, распределение сходства процессоров по портам, безопасный порог обратной записи дескрипторов Tx и опциональное отключение опроса статуса SFP.
- Добавлена настройка неуправляемых (unmanaged) устройств NetworkManager для лабораторных тестов, чтобы предотвратить сброс ручной конфигурации IP.
- Добавлено руководство по устранению неполадок для двухпортовых тестов: использование `iperf3 -B`, проверка раздельной маршрутизации, проверка сопоставления IRQ и рассмотрение быстро растущих ошибок `rx_crc_errors` как проблем физического уровня.

- Очистка сборки: удалены предупреждения GCC `-Wint-in-bool-context` при использовании `TCALL()`, возвращающих статус.

- Очистка сборки для LoongArch: подавление диагностики ассемблера GNU об отрицательном сдвиге вправо только для внешних модулей LoongArch.

- Исправление совместимости hwmon для ядер, где `hwmon_device_register()` устарел.
- Процесс регистрации теперь предпочитает: `hwmon_device_register_with_info()` -> `hwmon_device_register_with_groups()` -> `hwmon_device_register()`.
- Исправлены пути очистки hwmon для предотвращения удаления при `ERR_PTR` и безопасного сброса внутреннего состояния hwmon.
- Сохранены существующие атрибуты температуры sysfs драйвера (`temp0_input`, `temp0_alarmthresh`, `temp0_dalarmthresh`).
- Подтверждена работа в реальном времени на 5.15.148-tegra (Jetson AGX Orin): стабильная загрузка/выгрузка, линк поднимается, изменение MTU на 9000 работает.
- Проверена матричная сборка для ядер: 3.18.140, 4.1.52, 4.4.302, 4.9.337, 4.14.336, 4.19.325, 5.4.302, 5.10.250, 5.15.200, 6.1.163, 6.6.124, 6.12.71, 6.18.10.
